Privacy Policy

Last updated: July 10, 2026

PopDrop does not collect, track, or transmit any user data. Period.

1. No Data Collection

PopDrop does not use analytics, crash reporters, advertising SDKs, or tracking scripts. We don't know who you are, what files you transfer, or when you use the app. There are no user accounts, sign-ups, or external databases.

2. No Cloud Servers

All file transfers happen locally over your own Wi-Fi or Ethernet network. When someone sends a file to your Mac using PopDrop, the data travels directly between those two devices inside your local network. It never crosses the internet or touches any cloud storage.

3. Why PopDrop asks for macOS permissions

To work correctly, Apple requires PopDrop to declare specific system permissions in its sandbox:

  • Local Network Access (com.apple.security.network.server): PopDrop runs a small local HTTP server on port 8080 so other devices on your local network can connect and upload files.
  • Downloads Folder Access (com.apple.security.files.downloads.read-write): Incoming files are saved directly to your local ~/Downloads/PopDrop folder. PopDrop only writes new files you receive; it doesn't scan or upload your existing files.
  • Custom Folder Access (com.apple.security.files.user-selected.read-write): If you change your download folder in Settings, macOS grants PopDrop permission to write to that specific folder you selected.

4. Issues & Support

If you have any questions about this policy or how PopDrop works locally, you can open an issue directly at kiwiprojekt.github.io/issues.
